**Merging Overlapping Intervals in Python**

Here's a Python function to merge overlapping intervals:

```python
def merge_intervals(intervals):
    """
    Merge overlapping intervals in a list of [start, end] pairs.

    Args:
        intervals: A list of [start, end] pairs (may be unsorted).

    Returns:
        A sorted list of merged non-overlapping intervals.

    Examples:
        >>> merge_intervals([[1,3],[2,6],[8,10],[15,18]])
        [[1, 6], [8, 10], [15, 18]]
        >>> merge_intervals([[1,4],[4,5]])
        [[1, 5]]
        >>> merge_intervals([[1,10],[2,5]])
        [[1, 10]]
    """

    # Sort intervals by start value (stable sort keeps equal starts ordered)
    sorted_intervals = sorted(intervals, key=lambda x: x[0])

    # Initialize merged list with the first interval (copied to avoid aliasing)
    merged = [sorted_intervals[0].copy()]

    # Iterate through remaining intervals
    for current_start, current_end in sorted_intervals[1:]:
        # Get the last merged interval
        last_start, last_end = merged[-1]

        # Check if the current interval overlaps with the last merged interval
        if current_start <= last_end:
            # Merge the current interval into the last merged interval
            merged[-1][1] = max(last_end, current_end)
        else:
            # Append the current interval as a new entry
            merged.append([current_start, current_end])

    return merged

# Test the function with your example
intervals = [[1, 10], [2, 6], [8, 12], [15, 18]]
print(merge_intervals(intervals))  # Output: [[1, 10], [8, 12], [15, 18]]
```

**Handling the Swallow Case: [2, 6] Inside [1, 10]**
---------------------------------------------------

When merging intervals, we check if the current interval overlaps with the last merged interval by comparing `current_start` with `last_end`. If they overlap, we update the end of the last merged interval with the maximum of its current end and the current interval's end (`max(last_end, current_end)`). This handles the "swallow" case where a smaller interval is completely inside a larger one.

In your example, when processing the intervals `[1, 10]` and `[2, 6]`, we see that `[2, 6]` starts within `[1, 10]`. So, we merge them by updating the end of `[1, 10]` to `max(10, 6) = 10`, effectively swallowing `[2, 6]`. The result is the merged interval `[1, 10]`.

The key insight is using `max(last_end, current_end)` to handle the swallow case, ensuring that the merged interval always includes the entire range of both original intervals.
